# COMPLEXITY CLASSIFICATION CRITERIA

## VAGUE Query
A query is **VAGUE** when it:
- Contains ambiguous pronouns or references without clear context (e.g., "it", "this", "that", "these", "those", "them")
- Uses non-specific terms like "something", "things", "stuff"
- Lacks sufficient detail to understand what the user is asking about
- References previous context that is not provided (assuming conversational history)
- Contains unclear or undefined technical terms without explanation
- Is too short or fragmentary to determine intent (e.g., "explain this", "what about it?")
- Ask about 

**Examples of VAGUE queries:**
- "How does it work?" (what is "it"?)
- "Tell me more about this" (what is "this"?)
- "Explain that method" (which method?)
- "What are the benefits?" (benefits of what?)
- "Compare them" (compare what?)
- "Is this better than that?" (what are "this" and "that"?)
- "Can you elaborate on those techniques?" (which techniques?)

**When a query is VAGUE:**
- The system should ask for clarification
- Suggest the user provide more context or specify what they're referring to
- Cannot proceed with retrieval without additional information 

---

## SIMPLE Query
A query is **SIMPLE** when it:
- Asks about a single, well-defined concept, method, or topic
- Has one clear information need
- Can be answered by retrieving documents about one specific subject
- Does not require comparing, contrasting, or relating multiple entities

**Examples of SIMPLE queries:**
- "What is Transformer architecture?"
- "How does BERT work?"
- "What are the benefits of attention mechanisms?"
- "Define gradient descent"
- "What is the purpose of batch normalization?"

## COMPLEX Query
A query is **COMPLEX** when it involves:

### 1. **Comparison or Contrast**
- Comparing two or more methods, models, or concepts
- Examples: "Compare Transformer and RNN", "Differences between BERT and GPT"

### 2. **Multiple Independent Topics**
- Asking about multiple unrelated or semi-related things in one query
- Examples: "What is CNN and how does transfer learning work?", "What is RAG? What is RNNs?"

### 3. **Relationships or Dependencies**
- Asking how X affects/impacts/influences Y
- Asking about X in the context of Y
- Examples: "Impact of learning rate on model convergence", "Role of attention in machine translation"

### 4. **Multi-Part Questions**
- Questions with multiple components that need separate treatment
- Examples: "What are the advantages and limitations of LSTM?", "How does YOLO work and what are its applications?"

### 5. **Contextual or Conditional Queries**
- Queries that ask about something under specific conditions or contexts
- Examples: "Best practices for fine-tuning LLMs on small datasets", "Transformer performance in low-resource languages"

### 6. **Broad or Overly General Queries**
- Overly general questions that lack specificity but still have clear subjects
- Examples: "Tell me about deep learning", "What is AI?", "Explain neural networks"

### 7. **Sequential or Procedural Questions**
- Questions asking about processes, workflows, or step-by-step information
- Examples: "How to train and deploy a model?", "Steps to implement RAG system"

---

# EVALUATION GUIDELINES

## Assess Complexity
1. **First, check if the query is VAGUE** - Does it contain unclear references or lack necessary context?
2. Identify how many distinct information needs exist in the query
3. Check if the query requires retrieving documents about multiple topics
4. Determine if the query involves relationships between concepts
5. Evaluate if the query is specific enough for effective retrieval

## Provide Reasoning
- Be concise (1-2 sentences maximum)
- Clearly state why the query is vague, simple, or complex
- Mention the specific pattern or characteristic identified
